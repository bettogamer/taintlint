# COMBAT_LOG_EVENT can no longer be registered (12.0+)

**Rule SV012 — direct `COMBAT_LOG_EVENT[_UNFILTERED]` registration.** Severity: `error`.

## The problem this rule prevents

On WoW 12.0+ ("Midnight"), registering the classic combat-log events errors immediately:

```lua
frame:RegisterEvent("COMBAT_LOG_EVENT_UNFILTERED")  -- errors on 12.0+
```

The combat log was restructured as part of the secret-values overhaul: addons no longer
receive raw CLEU payloads via `RegisterEvent` + `CombatLogGetCurrentEventInfo`. The
replacement is the **`C_CombatLog`** namespace.

## Example that breaks in-game

```lua
local f = CreateFrame("Frame")
f:RegisterEvent("COMBAT_LOG_EVENT_UNFILTERED")   -- SV012
f:RegisterEvent("COMBAT_LOG_EVENT")              -- SV012
```

## How to fix

Migrate to `C_CombatLog`, and treat payload fields as potentially secret (guard with
`issecretvalue` before arithmetic/comparison/table-keying, as everywhere else on 12.0+).

If the addon supports multiple client flavors, gate by API presence rather than by version
number:

```lua
if C_CombatLog then
  -- 12.0+ path
else
  f:RegisterEvent("COMBAT_LOG_EVENT_UNFILTERED")  -- classic-era clients
  -- taintlint: allow SV012 (pre-12.0 client path, gated by C_CombatLog check)
end
```
