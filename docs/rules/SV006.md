# "attempt to index local 'aura' (a secret value)"

**Rule SV006 — indexing a possibly-secret table or value.** Severity: `error` (unconditionally
secret API), `warning` (conditionally secret, or via data-flow).

## The error this rule prevents

Seen in BugSack / the default error frame on WoW 12.0+ ("Midnight"):

```
attempt to index local 'line' (a secret value)
attempt to index a secret value (while execution tainted by 'MyAddon')
```

Some APIs return entire **secret tables** — most prominently the aura data from
`C_UnitAuras.GetAuraDataByIndex` / `GetAuraDataBySlot` / `GetAuraDataBySpellName`. Indexing one
(`aura.spellId`, `aura["duration"]`) throws. Method calls on secret strings
(`UnitName(u):lower()`) are the same operation and throw too.

## Example that breaks in-game

```lua
local spellId = C_UnitAuras.GetAuraDataByIndex("player", 1).spellId  -- SV006
local lowered = UnitName("target"):lower()                           -- SV006
```

Note: the player-unit heuristic does **not** apply here — aura restrictions cover the player's
own auras in combat.

## How to fix

Store first, guard with `issecrettable` (the table variant of `issecretvalue`), then use:

```lua
local issecrettable = issecrettable or function() return false end

local aura = C_UnitAuras.GetAuraDataByIndex("player", 1)
if aura and not issecrettable(aura) then
  print(aura.spellId)
end
```
