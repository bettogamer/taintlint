# "attempt to call a nil value (global 'issecretvalue')"

**Rule SV011 — `local issecretvalue = issecretvalue` without a pre-12.0 fallback.**
Severity: `info`.

## The error this rule prevents

Seen on **pre-12.0 clients** (Classic flavors, or any client where the secret-value API does
not exist) when running code written for 12.0+:

```
attempt to call a nil value (global 'issecretvalue')
attempt to call upvalue 'issecretvalue' (a nil value)
```

`issecretvalue` / `issecrettable` / `canaccessvalue` / `canaccesstable` only exist on 12.0+
clients. The common localization pattern caches a `nil` on older clients, and the guard itself
becomes the crash:

## Example that breaks on older clients

```lua
local issecretvalue = issecretvalue   -- SV011: nil on pre-12.0 clients

-- later:
if issecretvalue(hp) then ... end     -- attempt to call a nil value
```

## How to fix

The MRT-style shim — safe on every client, zero cost on 12.0+:

```lua
local issecretvalue = issecretvalue or function() return false end
local issecrettable = issecrettable or function() return false end
```

On pre-12.0 clients nothing is ever secret, so a constant-`false` fallback is semantically
correct, not just crash avoidance.
