# Secret values in BigWigs / DBM callback arguments

**Rule SV008 — boss-mod callback args operated without a surgical guard.** Severity: `warning`.

## The error this rule prevents

Seen mid-encounter on WoW 12.0+ when an addon consumes BigWigs or DBM events:

```
attempt to perform arithmetic on a secret value (while execution tainted by 'BigWigs')
attempt to perform arithmetic on local 'duration' (a secret number value, while execution tainted by 'DBM-Core')
```

Handlers registered for `BigWigs_*` / `DBM_*` messages run as the boss mod's callback, and on
12.0.x their arguments (duration, spellId, text…) **can arrive as secret values**. Operating
them unguarded throws in the middle of a boss fight.

## Example that breaks in-game

```lua
BigWigsLoader.RegisterMessage(myAddon, "BigWigs_StartBar",
  function(event, module, key, text, duration)
    local endTime = GetTime() + duration   -- SV008: duration can be secret
    bars[text] = endTime                   -- SV008: text can be secret (table key)
  end)
```

## How to fix — guard surgically, not broadly

Check `issecretvalue` **only on the args you actually operate on**, and degrade just that
action. A broad "discard the event if any arg is secret" guard is a false fix: real timers
routinely carry one secret arg among usable ones (and DBM vs BigWigs differ on which), so the
broad guard silently kills timers that would have worked.

```lua
local issecretvalue = issecretvalue or function() return false end

BigWigsLoader.RegisterMessage(myAddon, "BigWigs_StartBar",
  function(event, module, key, text, duration)
    if issecretvalue(duration) or issecretvalue(text) then return end  -- skip THIS bar only
    bars[text] = GetTime() + duration
  end)
```

Other proven degradations from ported addons: fall back to a default
(`if issecretvalue(arg2) then arg2 = 40 end`), or re-key a bar with non-secret data
(`"bw_" .. spellId .. "_" .. floor(duration)`).

The event name itself (first callback arg) is never secret — comparing it is safe and is not
flagged.

## Notes

- On 12.0.7, BigWigs bars arrive via the universal `BigWigs_StartBar` message (signature:
  `module, key, text, duration, icon, …`) — not `BigWigs_Timer`. If your handler only listens
  to `BigWigs_Timer`, ability bars never fire (pull/break timers use their own messages).
