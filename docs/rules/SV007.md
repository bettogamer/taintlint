# "attempt to call a secret value"

**Rule SV007 — calling a possibly-secret value as a function.** Severity: `error`
(unconditionally secret API), `warning` (conditionally secret, or via data-flow).

## The error this rule prevents

Seen in BugSack / the default error frame on WoW 12.0+ ("Midnight"):

```
attempt to call a secret value
attempt to call local 'fn' (a secret value, while execution tainted by 'MyAddon')
```

Calling a [secret value](https://warcraft.wiki.gg/wiki/Secret_Values) as a function is
forbidden in tainted code, like every other operation that would inspect or execute it.

## Example that breaks in-game

```lua
local result = C_UnitAuras.GetAuraDataByIndex("target", 1)()   -- SV007
```

In real code this usually appears indirectly: a callback slot or dispatch table gets populated
with a value that arrives secret in combat, and the call site explodes later.

## How to fix

Guard before calling:

```lua
local issecretvalue = issecretvalue or function() return false end

local fn = candidate
if type(fn) == "function" and not issecretvalue(fn) then
  fn()
end
```

(`type()` accepts secret values, so `type(fn) == "function"` is safe.)
