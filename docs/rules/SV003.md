# "attempt to concatenate a secret number value"

**Rule SV003 — concatenation / `tostring()` of a secretable API result.** Severity: `error`
(unconditionally secret API), `warning` (conditionally secret, or via data-flow).

## The error this rule prevents

Seen in BugSack / the default error frame on WoW 12.0+ ("Midnight"):

```
attempt to concatenate a secret number value
attempt to concatenate local 'hp' (a secret value, while execution tainted by 'MyAddon')
```

On tainted execution paths, using `..` or `tostring()` on a
[secret value](https://warcraft.wiki.gg/wiki/Secret_Values) throws — concatenation would reveal
the value, so it is forbidden like arithmetic and comparison.

## Example that breaks in-game

```lua
local label = "HP: " .. UnitHealth("player")     -- SV003
local text  = tostring(UnitHealth("player"))     -- SV003
```

## How to fix

`string.format` is explicitly allowed to receive secret values — the formatted string is
produced without your code ever inspecting the number:

```lua
local label = string.format("HP: %d", UnitHealth("player"))
```

Or hand the value straight to the UI, which also accepts secrets:

```lua
myFontString:SetText(UnitHealth("player"))
myFontString:SetFormattedText("HP: %d", UnitHealth("player"))
```

If you truly need a plain string, guard first:

```lua
local issecretvalue = issecretvalue or function() return false end
local hp = UnitHealth("player")
local label = issecretvalue(hp) and "HP: ??" or ("HP: " .. hp)
```
