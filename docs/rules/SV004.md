# "attempt to get length of a secret value"

**Rule SV004 — `#` length operator on a secretable API result.** Severity: `error`
(unconditionally secret API), `warning` (conditionally secret, or via data-flow).

## The error this rule prevents

Seen in BugSack / the default error frame on WoW 12.0+ ("Midnight"):

```
attempt to get length of a secret value
attempt to get length of local 'name' (a secret value, while execution tainted by 'MyAddon')
```

The length operator `#` inspects a [secret value](https://warcraft.wiki.gg/wiki/Secret_Values)
(string length leaks information about the content), so tainted code may not apply it.

## Example that breaks in-game

```lua
local nameLen = #UnitName("target")   -- SV004: "target" can be an arbitrary unit
```

## How to fix

Guard first, or avoid measuring the value:

```lua
local issecretvalue = issecretvalue or function() return false end
local name = UnitName("target")
local nameLen = issecretvalue(name) and 0 or #name
```

Note the unit-token nuance: identity restrictions (`SecretWhenUnitIdentityRestricted`) never
apply to player-controlled units, so taintlint does **not** flag literal `player`/`pet`/
`party1..4`/`raid1..40` tokens:

```lua
local nameLen = #UnitName("player")   -- not flagged: never secret
local other   = #UnitName("raid7")    -- not flagged: raid units are players
```
